Coursework & Notes

Since learning of the terminal in my computer, I was a bit obsessed. For the first few months, I was fascinated by it and almost all my conversations and studies were centered around it. The first programming language I was introduced to was Bash. My instructors showed me how to use it in the terminal, assuring me that I would not be able to do something one of them could not undo. With that in mind, I dubbed myself a basher and I began bashing.

From my interactions with the terminal and Bash, I became intrigued albeit frustrated by programming and the languages. I spoke with my instructors and they explained that programming languages are their own kind of languages. This prompted me to dig deeper and I connected that to my personal relationship to the English language. I thought I would give myself a gift that semester and design a programming language that was more intuitive to me.

I began with a tutorial one of my instructors asked me to do to get familiar with Bash:

  • How to run an interactive script in the terminal...
    • Create a new text file in Sublime Text.
    • Add the following lines of code to that file:
    • echo "What is your name?"
    • read their_name
    • echo "Hi $their_name!"
    • Save the file. You can call it whatever you'd like, but should end the file name with .sh Let's say you call it file.sh
    • Now, go to your terminal. Make sure that you cd into the directory where you saved the file (e.g. if you saved it to the Desktop, cd into Desktop) and then run the command:
    • source file.sh
    • You should be asked what your name is in the terminal and be able to get a response to it.

This was the beginning of what I thought would be the best final project and gift I would ever give - a language that would be useful in programming and one that I would understand. I had decided to call my language Gittish and it was to combine Git and English into an intuitive language for those who may struggle with either or both of them. One would be able to access Gittish via running the command, source gitti.sh in the terminal.

The greatest hurdle I would face was in gaining a thorough understanding of Git, the terminal, and Bash. I needed to learn why we need them, how they got here, how they work, how they work together, what happens if we didn't have them, what are the most popular uses for them...

My instructors sent videos about git commands and push/pull requests. Along with extensive independent study, I met almost weekly, one-on-one, with an instructor to explore the new tech and languages being introduced. I attended all the studios that I could.

During class, we were introduced to all kinds of technology and programming capabilities. I downloaded and set up my first text editor, Sublime Text to practice the basics of HTML/CSS/JS from the course demo. I set up my AWS S3 account although I had no idea what to do with an account like that. There was time during class devoted to testing out a Tessel running commands in the terminal as you can find in this copy of my terminal history. We explored other people's digital gifts through forking their repositories like, Letterpress and a-story-in-pomes. We also got to play with an L3D Cube through an in-browser interface, 3D-GIF. All of these experiences led me to think of more digital gifts and I would jot down in my notebook.
